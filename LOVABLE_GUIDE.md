# Lovable UI 디자인 요청서

네이버 플레이스 순위 분석 서비스입니다. 아래 2개 페이지의 UI를 예쁘게 디자인해주세요.

---

## 공통 사항

- 헤더: 로고 + "순위 조회", "순위 추적" 2개 메뉴
- 모던하고 깔끔한 디자인
- 모바일 반응형

---

## 페이지 1: 순위 조회 (메인 페이지)

### 기능 흐름
**1단계 → 2단계 → 결과** 순서로 진행

### 1단계: 업체 검색
- 제목: "1단계: 내 업체 찾기"
- 입력창: 업체명 검색
- 검색 버튼
- 검색 결과: 업체 목록 (업체명, 카테고리) - 클릭하면 선택됨

### 2단계: 키워드 입력
- 제목: "2단계: 순위 조회"
- 선택된 업체 표시 (업체명, 카테고리)
- "다시 검색" 버튼
- 입력창: 검색 키워드 (쉼표로 구분, 예: "강남 맛집, 강남역 맛집")
- 입력창: 유입수 (선택사항, 숫자)
- "순위 조회하기" 버튼
- 로딩 중일 때: "분석에 약 1분 정도 소요됩니다" 안내 표시

### 결과 화면

#### 1. 업체 정보 카드
- 업체명, 카테고리
- 방문자 리뷰 수
- 블로그 리뷰 수
- 플레이스 지수

#### 2. 키워드별 순위 카드
- 키워드마다 카드 형태로 표시
- 각 카드: 키워드명, 순위 뱃지 (1-3위: 금색, 4-10위: 녹색, 11-30위: 파랑, 그 외: 회색)
- 전체 검색 결과 수 표시

#### 3. 내 업체 분석 카드
- 키워드명 표시
- 현재 순위 (큰 숫자)
- 종합 점수 (큰 숫자, 100점 만점)
- 1위인 경우: "축하합니다! 1위입니다" 메시지
- 특정 항목에서 1위인 경우: "OO에서 1위입니다! 이 항목은 현재 상태를 유지하세요" 메시지

#### 4. "1위가 되려면" 섹션 (1위가 아닐 때만)
- 점수 현황: "현재 72.5점 → 1위 85.3점 (12.8점 부족)"
- 우선순위별 개선 항목 리스트:
  - 각 항목: 순위, 아이콘, 항목명, 1개당 점수
  - 현재 개수 → 필요 개수 (+예상 점수)
  - 이 항목만으로 1위 가능한 경우: 녹색 배경, "이 항목만 달성하면 1위!" 메시지
  - 불가능한 경우: "최대 +X점 가능 (다른 항목도 필요)" 메시지
- 이미 1위인 항목: 녹색 배경, "1위 (000개) - 이 항목은 현재 상태를 계속 유지하세요"

#### 5. 경쟁 업체 테이블
- 테이블 형태
- 컬럼: 순위, 업체명, 방문자 리뷰, 블로그 리뷰, 최신(1주), 상세 링크
- 내 업체는 하이라이트 처리

## 페이지 2: 순위 추적

### 상단
- 제목: "순위 추적"
- 설명: "중요한 키워드의 순위 변동을 지속적으로 모니터링하세요"

### 액션 버튼
- "전체 새로고침" 버튼
- "키워드 추가" 버튼 → 다이얼로그 열림

### 키워드 추가 다이얼로그
- 입력창: 플레이스 URL
- 입력창: 업체명 (선택)
- 입력창: 키워드
- "추가하기" 버튼

### 저장된 키워드 테이블
- 컬럼: 업체명, 키워드, 현재 순위, 지수, 방문자리뷰, 블로그리뷰, 액션
- 순위 뱃지: 1-3위 금색, 4-10위 기본, 그 외 회색
- 지수: 70+ 녹색, 40-70 노랑, 40 미만 빨강
- 액션 버튼: 히스토리, 새로고침, 삭제

### 확장 가능한 행
- 행 클릭 시 아래로 펼쳐짐
- "최근 7일 데이터" 표시
- 가로 테이블: 날짜별 순위, 방문자리뷰, 블로그리뷰, 지수

### 히스토리 다이얼로그
- 제목: "순위 히스토리"
- 업체명 - 키워드 표시
- 라인 차트: X축 날짜, Y축 순위 (역순, 1위가 위)

### 빈 상태
- 키워드가 없을 때: 아이콘 + "저장된 키워드가 없습니다" + "키워드를 추가하여 순위 변동을 추적하세요"

## 디자인 참고사항

- 카드 기반 레이아웃
- 뱃지로 상태 표시 (순위, 점수 등)
- 테이블은 깔끔하게
- 로딩 상태 표시 (스피너)
- 성공/에러 토스트 메시지
- 버튼: Primary(파랑), Secondary(회색), Destructive(빨강-삭제용)
